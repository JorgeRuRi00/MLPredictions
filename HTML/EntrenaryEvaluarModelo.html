from sklearn.metrics import mean_absolute_error, mean_squared_error, r2_score
#Definimos una funcin genrica para evaluar los modelos, as podemos comparar varios de manera sencilla y se simplifica el cdigo
def evaluate_model(y_true, y_pred, model_name):
    mae = mean_absolute_error(y_true, y_pred)
    mse = mean_squared_error(y_true, y_pred)
    r2 = r2_score(y_true, y_pred)
    print(f'{model_name} - MAE: {mae}, MSE: {mse}, R: {r2}')
    return {'Model': model_name, 'MAE': mae, 'MSE': mse, 'R': r2}

De los resultados obtenidos, la MAE mide el error promedio de las predicciones en las mismas unidades que las variables originales. Los modelos con menor MAE son mejores. El support Vector Regression tiene el menor MAE (0.1909), seguido de cerca por el Gradien Boosting Regressor (0.1930). Por otro lado, la MSE mide el promedio de los errores al cuadrado. Es ms sensible a grandes errores debido a la cuadratura. Por ltimo, el R^2 mide la proporcin de la varianza en la variable dependiente que es predecible a partir de las variables independiente. Un valor de R^2 cercano a 1 indica un modelo muy bueno. El Gradient Boosting Regressor tienen un R^2 positivo (0.0095), lo qeu sugiere que es el mejor modelo en trminos de explicacin de la varianza, aunque el valor resulta ser bastante bajo.

Conclusin

Gradient Boosting Regressor parece ser el mejor modelo en trminos de MAE, MSE y R, aunque los valores absolutos del R sugieren que todava hay espacio para mejorar el modelo. Support Vector Regression tambin muestra un buen rendimiento, especialmente en trminos de MAE. Los modelos Decision Tree Regressor y Random Forest tienen los peores desempeos, con valores de R negativos que indican un mal ajuste a los datos.

El haber obtenido valores de R^2 negativos indica que los modelos no son muy eficientes aun. Un R^2 negativo indica de hecho, que el modelo es peor que una lnea horizontal que predice el valor medio de los datos. El MAE, aunque cercano a 0 aun es mejorable. El MSE si que da valroes buenos muy cercanos a cero, indicando que los errores grandes no son comunes.

Los motivos de estos resultados de estas predicciones aun algo deficientes puede deberse a la simplicidad del modelo, es decir, tenemos modelos demasiado simples para captar la complejidad de los datos (Los modelos lineales o con parmetros preestablecidos pueden no ser suficientes).

La normalizacin puede afectar a la interpretacin de R^2 si los datos no esta adecuadamente ajustados para el modelo. Adems, el uso de dos caractersticas puede estar limitando las predicciones.

Por ltimo, los datos pueden tener relaciones no lineales que no son capturadas por modelos lineales o sin una buena caracterizacin de hiperparmetros.

from sklearn.model_selection import RandomizedSearchCV
from scikeras.wrappers import KerasRegressor
from tensorflow.keras.layers import Input

# Definir la funcin para crear el modelo
def create_model(neurons=100, dropout_rate=0.2, optimizer='adam'):
    model = Sequential()
    model.add(Input(shape=(X_train.shape[1],)))
    model.add(Dense(neurons, activation='relu'))
    model.add(Dropout(dropout_rate))
    model.add(Dense(neurons, activation='relu'))
    model.add(Dropout(dropout_rate))
    model.add(Dense(1))
    model.compile(loss='mean_squared_error', optimizer=optimizer, metrics=['mean_absolute_error'])
    return model

# Crear el modelo KerasRegressor
model = KerasRegressor(model=create_model, verbose=0)

}
